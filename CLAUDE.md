## Architecture

This is **GSTOK**, a React 19 + TypeScript + Vite single-page app. The UI is built on **Material UI (MUI)** and started from MUI's marketing-page and admin-dashboard example templates â€” a lot of component structure and comments still reflect that origin.

### Routing

Routing uses **TanStack Router** with file-based routes under [src/routes/](src/routes/):
- [src/routeTree.gen.ts](src/routeTree.gen.ts) is auto-generated by the `@tanstack/router-plugin` Vite plugin (configured in [vite.config.ts](vite.config.ts) with `autoCodeSplitting: true`). Never edit it by hand â€” it regenerates whenever route files change while `vite`/`npm run dev` is running.
- [src/routes/__root.tsx](src/routes/__root.tsx) is the root layout: wraps everything in `AppTheme` + `CssBaseline` and renders `TanStackRouterDevtools`.
- [src/routes/admin/route.tsx](src/routes/admin/route.tsx) defines the `/admin` layout route, rendering `AdminTemplate` as the shell for all nested admin pages (`clientes`, `produtos`, `pedidos`, `usuarios`, `configuracoes`).
- Route files are thin: they call `createFileRoute(path)({ component })` and delegate the actual UI to components in `src/features/**` or `src/templates/**`.

### Feature/template/theme split

- [src/features/](src/features/) â€” page-level feature UIs (e.g. `home`, `login`), each with its own `components/` folder. Routes import from here.
- [src/templates/AdminTemplate/](src/templates/AdminTemplate/) â€” the admin dashboard shell (side menu, app bar, header, breadcrumbs, etc.), mirrored from MUI's dashboard template.
- [src/theme/](src/theme/) â€” MUI theme setup: `AppTheme.tsx` builds the theme from `themePrimitives.ts` and per-area `customizations/*` (dataDisplay, feedback, inputs, navigation, surfaces), plus `customTheme.ts` and color-mode selection components.
- [src/components/layout/PageLayout.tsx](src/components/layout/PageLayout.tsx) â€” shared page wrapper (title + action button + content slot) intended for use inside admin pages; note most `admin/*.tsx` routes are still stub placeholders (`<div>Hello "..."!</div>`) that haven't adopted it yet.

### Conventions to be aware of

- Path aliases are not configured â€” imports use relative paths (`../../theme/AppTheme`).
- Formatting is enforced by Prettier with no semicolons, single quotes, trailing commas ([.prettierrc](.prettierrc)); however code style is inconsistent across files (some use `import {X} from` without spaces and semicolons â€” remnants of the original MUI template vs. newer app code). Match the surrounding file's style rather than the global default when in doubt.
- Admin routes/UI text are in Portuguese (`clientes`, `produtos`, `pedidos`, `usuarios`, `configuracoes`); keep new admin-facing strings in Portuguese for consistency.
- API base URL comes from `VITE_API_URL` (see [.env.development](.env.development)); no HTTP client/data-fetching layer exists yet.
- `src/routes/admin/ususarios.tsx` is a stray misspelled duplicate of `usuarios.tsx` â€” be aware both currently exist as separate routes.
